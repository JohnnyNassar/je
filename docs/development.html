Lessons learned

OPcache vs deploys

PHP-FPM had opcache.validate_timestamps=0 in its config, so simply replacing PHP files left old bytecode in memory and made fixes look like they had no effect. All deploys now run systemctl reload php8.3-fpm as the last step.

bash + bcrypt

Passing a bcrypt hash through a bash variable into mariadb -e "UPDATE ... password=\"$HASH\"" mangles the $ characters of the hash. Fix: do the hashing and the UPDATE inside a single PHP script (or use Eloquent's hashed cast).

Subdirectory + Livewire

Running Laravel from a sub-path (http://localhost/joreption) confuses Livewire's default URL detection. Solution: override Livewire::setUpdateRoute() / setScriptRoute() in AppServiceProvider — but only when the path prefix is non-empty (don't apply on production where there's no subpath, otherwise it double-registers and breaks).

Filament + production

Filament 3 defaults to denying all panel access when APP_ENV !== 'local'. The User model must implement FilamentUser::canAccessPanel() to allow login.

Static asset regex + virtual files

Nginx regex blocks like location ~* \.(css|js|jpg|...)$ will try to serve them as static files and 404 when they don't exist on disk. Add try_files $uri /index.php?$query_string; to fall through to Laravel for virtual assets (Livewire's /livewire/livewire.js, etc.).

First-connect SSH from Windows

OpenSSH on Windows doesn't have sshpass. Use PuTTY's plink -pwfile {file} -hostkey "SHA256:..." to bootstrap key-only SSH from a password-only state. Computing the host key fingerprint via ssh-keyscan ahead of time avoids the interactive first-connect prompt.

WhatsApp export is messy

<Media omitted> outnumbers actual file references; product names are buried in mixed Arabic+English ad-copy; prices use multiple syntaxes; admin promo lines like 25JD end up as product "names" without manual review. Plan on importing as drafts, not active, and reviewing in the admin.

Pricing dialect matters

السعر is the actual selling price; سعره بالسوق is the market reference price (skip); Online $X is a reference price (skip). Got bitten by accepting any number followed by دينار as the price — fixed by ordering regex patterns from most-specific to least.

Filament modal <script> tags don't execute

When Filament injects modal content via innerHTML, inline <script> blocks are silently skipped (browser security). Register Alpine components globally via a standalone JS file loaded on every panel page (a renderHook(panels::head.end, …)).

HTML attribute quotes vs JS string quotes

Nested " strings inside x-data="..." corrupt the attribute, and with @js() emitting double-quoted JSON it's a constant battle. Lesson: don't write multi-line JS inline in HTML — use Alpine.data() registrations.

Filament FileUpload state ≠ plain string

Setting $this->data['image_path'] = 'foo.jpg' directly crashes BaseFileUpload with "foreach() argument must be of type array, string given" on re-render. Fix: route updates through $this->form->fill($merged) so the component's hydration runs.

composer dump-autoload for new classes on prod

Adding a new app/Concerns/Foo.php and rsyncing it isn't enough — Composer's classmap doesn't know about it until composer dump-autoload runs. Symptom: Method Foo::bar not found from a $wire.call.

Git as root inside a www-data tree

git init triggers "dubious ownership" since git 2.36 — fix with git config --global --add safe.directory /var/www/joreption. Also set -e + git remote remove on a missing remote aborts the script; use git remote remove origin 2>/dev/null || true.

git reset --hard is safe for prod sync

It removes tracked-but-modified files but leaves untracked + gitignored files alone — so .env, vendor/, node_modules/ and uploaded images survive, as long as they're in .gitignore. But Filament's published public/{js,css}/filament/ disappear if gitignored — re-run php artisan filament:upgrade every deploy.

Alpine x-model + immediate form.submit() race

Setting a bound property then calling native form.submit() on the same synchronous line submits the old value — x-model flushes on a microtask, after the submit. Wrap the submit in $nextTick(() => …). (This is why the cart +/− "did nothing".)

Editing an already-run migration doesn't re-run it

A migration recorded in the migrations table won't re-execute on migrate. To fix a bad/empty one across environments, add a new idempotent migration (Schema::hasColumn guards) rather than editing the old file in place.

Trait-based canAccess() gates Filament cleanly

Overriding canAccess(): bool on a Resource or Page both removes it from navigation and 403s on direct URL — one trait (AdminOnly) covers both. Verify trait composition by force-loading the class (class_exists) since php -l won't catch a trait-method collision.

Derived columns via model events

Keeping products.stock as the sum of variant stock is done in ProductVariant's saved/deleted events calling syncStockFromVariants() with saveQuietly() — avoids re-firing parent events and keeps every existing stock query working. Filament's ->relationship() repeater fires each child's events on save, so the roll-up just works from the admin form too.

Filament's default primary is Amber

To theme the admin, set panel->colors(['primary' => Color::hex(...)]) — applied as runtime CSS variables, so config:clear + a php-fpm reload is enough; no npm run build. A real "management section" also wants its own navigationGroup entry, not folding into general Settings.

Page-scoped widgets stay off the dashboard

To show stat/chart widgets on one page but keep them off the dashboard, put them outside app/Filament/Widgets/ (so discoverWidgets skips them) and reference them from the page's getHeaderWidgets().

Email test falls back to the From address

The "Send test email" action sends to the From address when no admin recipient is set — a blank "admin email" silently sends mail to a no-mailbox address (noreply@) that never lands in an inbox. Set a real recipient.

Audit causer gating + per-environment settings

Keying activity logging on auth('web')->user() turns it from a write firehose into a clean "who-did-what in the admin" trail — storefront stock decrements and customer order creation are intentionally excluded. Like email, the GA4 ID and Coming Soon toggle live in each environment's DB — set them on the live admin, not local.

Two separate role systems

Admin User.role (super_admin/admin/staff, web guard) is distinct from storefront Customer.tier (regular/vip/wholesale, customer guard) — "customer" is not a User role. Redefining isAdmin() as "admin-or-super_admin" let the tier split land with zero edits to existing gates.

Safe prod E2E test

A rolled-back DB::transaction + Mail::fake() lets you exercise the real cart/order/loyalty code on production with zero residue and no accidental emails — far safer than fighting CSRF + Coming Soon over HTTP. (Watch out: a WHERE name LIKE 'TEST%' cleanup check can match pre-existing data — confirm by the rolled-back row count, not a name match.)

Find Filament's real spacing in the DOM

Don't guess class names. Row height came from .fi-ta-text's py-4 and the page wrapper's py-8 / gap-y-8, not the obvious .fi-ta-cell. One computed-style walk in a headless browser found every contributor; blind CSS guesses had no-opped. Density needs no build step — a per-surface CSS file + a single :root { font-size } knob compacts everything proportionally.

Two auth guards keep drafts safe

Staff/admin = web guard (users), shoppers = customer guard (customers). Gating draft preview on auth('web')->check() lets the team see drafts without ever exposing them to logged-in customers or guests. And keeping image_path as the cover while layering a gallery array on top meant no consumer (grid/cart/orders/table) needed touching.
