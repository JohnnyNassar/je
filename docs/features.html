Loyalty points & customer tiers

Loyalty points program

  • Per-customer points balance + full earn / redeem ledger
  • Earning — credited automatically when an order is delivered
  • Guests accrue too (account auto-created from phone)
  • VIP customers earn at a configurable multiplier (e.g. ×2)
  • Redeeming at checkout — stacks with coupons, capped by min-redeem
  • Promotions — double/triple or bonus points, date-windowed
  • "You'll earn X points" hint at checkout · off by default

Customer tiers

  • Per-customer tier badge: Regular / VIP / Wholesale
  • Wholesale — configurable % off every unit price (cart → order)
  • VIP — points-earning multiplier
  • Filterable on the Customers list
  • Tier discount % & VIP multiplier set in Settings / Loyalty

Orders & customers

  • Points earned / redeemed columns on orders
  • Customers — name, phone (copyable), city, tier badge

Categories & coupons

  • 2-level hierarchy — Parent category selector (capped at 2 levels)
  • Bilingual name (auto-slug), drag-reorder, active toggle, count badge
  • Seeded standard taxonomy — 10 parents / 40 sub-categories
  • Coupons — percent or fixed, min order, usage limit, date window

  • Super-admin-only: Staff, Settings, Loyalty settings, Activity log
  • Admin runs Orders, Customers, Coupons, Loyalty
  • Staff manager (create/edit; can't delete own account)
  • "Can view cost prices" per-user toggle for a single staff member

Media Library

  • WordPress-style image manager (/admin/media-library)
  • Stats row: total files, disk usage, orphan count
  • 6-column thumbnail grid from products/ + hero/
  • Filters: search, all / used / orphans, sort by date/size/name
  • Tiles show "used" / "orphan" badges; modal preview + copy URL
  • Delete-if-orphan only (in-use files protected); 48 per page

Settings

  • Currency (code / symbol / before-or-after)
  • Admin WhatsApp number · wholesale discount %
  • Coming Soon mode + custom EN/AR headlines
  • Google Analytics (GA4) Measurement ID
  • Landing hero — upload (cropper), media library, or product image
  • Cached key/value store (invalidated on save)

Activity log (super-admin-only)

  • Audit trail of create / update / delete on products, orders, coupons, promotions, categories, settings, staff
  • Each entry: who, before → after values, timestamp
  • Auth events — admin/staff + customer logins/logouts, failed admin logins
  • Storefront order placements logged (attributed or guest)
  • Records client IP + device / user-agent; secrets redacted
  • Read-only, filterable — lightweight built-in (no package)

Profile & auth

  • Admin profile page (change name / email / password)
  • FilamentUser contract enforced in production
  • Separate web (admin) and customer guards
  • Both guards can be logged in at once; sign-ins logged with IP + UA

Infrastructure & tooling

  • Contabo Cloud VPS 10 — 4 vCPU / 8 GB / NVMe
  • Ubuntu 24.04 LTS, Asia/Amman timezone
  • Nginx + PHP 8.3-FPM + MariaDB 10.11 + Redis 7
  • HTTPS via Let's Encrypt (auto-renew via certbot.timer)
  • SSH: ed25519 key-only · UFW (22/80/443) · fail2ban
  • Daily DB backup at 03:00, 14-day retention · /_adminer (basic auth)
  • Push-based deploy via GitHub — joreption-deploy.sh over SSH
  • Composer + npm run build + filament:upgrade + migrate + cache
  • Redis for sessions + cache · OPcache (PHP-FPM reload on deploy)
  • WhatsApp importer — php artisan whatsapp:import {path}
  • 38 products imported as drafts; price/stock regex + UTF-8 cleanup
  • Session Cart service · money_format() · ImageResizer GD helper
