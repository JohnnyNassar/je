Email Setup for joreption.com

How to get real @joreption.com mailboxes and let the store send email.

Where things stand

The website runs on a Contabo VPS, and the domain's DNS is at Namecheap. Right now the domain has no MX records and no email hosting, so this is a one-time setup. Email hosting is a separate service from the website — adding it does not touch the site.

Don't host email on the Contabo VPS. It's technically possible, but a bad idea: Contabo IPs are often on spam blocklists, outbound port 25 is usually blocked/throttled, and running a secure mail server is a constant maintenance burden — your mail would mostly land in spam. Use a dedicated mail host (below) instead.

A Namecheap Private Email — mailboxes + SMTP

Easiest path, because the domain already lives at Namecheap, so the MX records auto-configure.

1 · Buy Private Email

  1. Log in to namecheap.com → Domain List → joreption.com.
  2. Open the Private Email product and buy a plan for the domain (the Starter / 1-mailbox plan is enough to start).

2 · Let Namecheap add the DNS records

Because DNS is managed by Namecheap, it will offer to auto-add the MX + SPF records — accept that. In the Private Email dashboard, also turn on DKIM and add the TXT it shows. If you ever add them manually (Domain → Advanced DNS):

TypeHostValuePriority
MX@mx1.privateemail.com10
MX@mx2.privateemail.com10
TXT (SPF)@v=spf1 include:spf.privateemail.com ~all—
TXT (DKIM)default._domainkeyshown in the Private Email dashboard—

DNS changes take anywhere from a few minutes to a couple of hours to take effect.

3 · Create a mailbox

  1. In the Private Email dashboard, create a mailbox such as orders@joreption.com (or info@, noreply@) and set a password. Keep that password handy.

4 · SMTP settings (you'll enter these in the platform)

FieldValue
Hostmail.privateemail.com
Port465 (SSL) — or 587 (TLS)
Usernamethe full email, e.g. orders@joreption.com
Passwordthat mailbox's password
EncryptionSSL for 465, TLS for 587
From addressthe same mailbox

→ Plug it into the platform

  1. Admin → Notifications → Email section.
  2. Turn on Enable email.
  3. Set From name (e.g. Joreption) and From address (your mailbox).
  4. Enter the SMTP host / port / username / password / encryption from the table above.
  5. Click Save, then the "Send test email" button at the top of the page.
  6. Check the inbox you sent the test to — if it arrives, you're done. Password-reset emails will now work too.
If the test fails, the on-screen error tells you why. The usual culprits: a mismatched port/encryption pair (use 465 + SSL or 587 + TLS), or a typo in the username/password.

B Resend — free, send-only (alternative)

If you only need to send (no inbox), this is free and quick.

  1. Create an account at resend.com and add the domain joreption.com.
  2. Add the DNS records Resend shows you at Namecheap → Advanced DNS (an SPF TXT, a DKIM TXT, and a return-path record). Wait until Resend shows Verified.
  3. SMTP: Host smtp.resend.com, Port 465 (SSL) or 587 (TLS), Username resend, Password = your Resend API key. From: any @joreption.com.
  4. Plug into the platform exactly as in the section above.

✓ When you're ready

Once you've added the DNS records, let your developer know — the MX / SPF / DKIM can be checked for propagation, then the test send can be run from the admin to confirm password-reset emails work end to end.
